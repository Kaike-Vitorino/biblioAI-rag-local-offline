# UI Audit Agent

Este documento define como agentes devem auditar visualmente a interface.

O objetivo é evitar que mudanças de UI sejam aceitas sem validação real.

---

# REGRA PRINCIPAL

Nenhuma alteração de UI é considerada concluída sem inspeção visual.

---

# FLUXO DE AUDITORIA

Sempre executar:

1. rodar aplicação local
2. abrir navegador
3. navegar fluxos principais
4. observar layout
5. identificar problemas
6. corrigir
7. validar novamente

---

# ÁREAS A INSPECIONAR

## Layout

Verificar:

- alinhamento
- uso do espaço
- posicionamento dos elementos

---

## Legibilidade

Verificar:

- tamanho da fonte
- contraste
- largura da coluna de leitura

---

## Interatividade

Testar:

- botões
- navegação
- controles

---

## Responsividade

Redimensionar tela e observar:

- layout quebrando
- elementos sobrepostos
- conteúdo invisível

---

# INSPEÇÃO DO PDF VIEWER

Sempre verificar:

- página centralizada
- highlight correto
- navegação funcional
- zoom funcionando
- seleção de texto funcionando

---

# CHECKLIST

Antes de aceitar a UI:

- [ ] layout equilibrado
- [ ] contraste adequado
- [ ] leitura confortável
- [ ] elementos alinhados
- [ ] highlight visível
- [ ] página bem posicionada
- [ ] interface responsiva

---

# REGRA FINAL

Se a interface:

- parece improvisada
- dificulta leitura
- desperdiça espaço

então ela precisa ser corrigida.
